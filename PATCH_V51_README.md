# V51 覆盖文件补丁：回测配置缓存 + 蛋卷最新估值日期对齐

覆盖文件：

- `app.py`
- `static/app.js`

## 修复内容

1. 【回测配置】表单增加本地持久化缓存。
   - 开始日期、结束日期、操作周期、手续费、滑点、估值参与方式、固定估值假设、导出选项都会保存到浏览器 localStorage。
   - 刷新页面或切换回仓位助手再进入回测页，配置不会丢。

2. 蛋卷历史估值序列补入 detail 最新估值点。
   - `pe_history/pb_history/roe_history` 可能比 detail 页面慢一个交易日。
   - 现在会读取 `djapi/index_eva/detail/{code}` 的 `ts` 作为【估值页面最新日期】。
   - 如果 detail 最新日期晚于历史曲线，会把 detail 的 PE/PB/ROE 补进历史序列，然后重新自算 PE/PB 百分位。

3. 回测指标里的日期对比更准确。
   - 【估值序列最新日期】现在可对齐蛋卷页面最新日期，例如 `2026-06-05`。
   - 不再因为历史曲线最后点是 `2026-06-04` 而误以为缺一天。

## 注意

- 历史回测仍然遵守避免未来函数：回测某一天只会使用该日及以前的估值数据。
- detail 最新点只会在其日期不晚于回测日时被使用；如果回测区间早于 detail 日期，不会污染过去回测。
