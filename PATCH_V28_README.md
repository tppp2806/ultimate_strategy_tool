## 本次修复

1. 修复“显示持有/观望，但建议金额仍出现小额买入/卖出”的问题。
   - 当调整幅度低于 0.5% 执行阈值时，目标仓位会同步归回当前仓位。
   - 结果会显示“维持 X% / 不操作”。

2. 修复“目标仓位低于当前仓位，但内部动作仍是观望”的问题。
   - 例如：未站上 200 日线，当前仓位过高，系统目标降到防守仓位时，会明确显示为“减仓”。

3. 修复“目标仓位接近 0%，但内部动作仍是减仓”的一致性问题。
   - 现在目标接近 0% 时，内部动作会同步为“清仓”。

## 自测场景

我用当前后端计算函数跑了以下场景，重点检查：动作、目标仓位、建议金额、仓位模式、标的类型是否一致。

| 场景 | 期望 | 实测结果 |
|---|---|---|
| 强多头 + 高 PE + 暂无买点 + 当前 10% | 不追买，维持 | 持有，维持 10%，不操作 |
| 纯交易仓 + 强多头 + 暂无买点 + 当前 10% | 不追买，维持 | 持有，维持 10%，不操作 |
| 强多头 + 回踩不破 + 当前 10% | 按风险上限加仓 | 加仓 +20%，目标 30% |
| 强多头突破但 PE 高 + 当前 50% | 不为 0.2% 小差额交易 | 持有，维持 50%，不操作 |
| 纯交易仓 + 触发初始止损 + 当前 30% | 清仓 | 清仓 -30%，目标 0% |
| 底仓模式 + 触发初始止损 + 高估 + 当前 30% | 可降到 0% | 清仓 -30%，目标 0% |
| 底仓模式 + 触发初始止损 + 不高估 + 当前 30% | 只保留极低防守仓 | 清仓 -25%，目标 5% |
| 触发初始止损但当前 3% 低于防守仓 | 不补仓、不追加卖出 | 持有，维持 3%，不操作 |
| 跌破 20 日线 + 当前 50% | 减仓一部分 | 减仓 -12%，目标 37.5% |
| 跌破 20 日线但当前 10% 低于防守仓 | 不补仓、不追加卖出 | 持有，维持 10% |
| 盈利 3R + 当前 80% | 分批止盈 | 止盈 -40%，目标 40% |
| 盈利 2R 但无风险形态 | 不急着止盈 | 持有，维持 50% |
| 盈利 2R + 长上影 | 止盈一部分 | 止盈 -16%，目标 33.5% |
| 空头趋势 + 当前 30% | 可清到 0% | 清仓 -30%，目标 0% |
| 未站上 200 日线 + 当前 30% | 降到防守状态 | 减仓 -10%，目标 20% |
| 强多头回踩但无止损距离 | 禁止新增仓 | 持有，维持 10%，不操作 |
| 强多头回踩但止损很远 | 小幅加仓 | 加仓 +5%，目标 15% |
| 防守/均衡/进攻三策略同场景 | 防守最慢，进攻最快 | 防守 +15%，均衡 +20%，进攻 +25% |
| stock/etf/fund/index 同信号 | 不按类型封顶 | 输出一致，均为加仓 +20% |

## 结论

- 当前系统的主要决策逻辑是可自洽的：趋势决定是否可做，入场/退出信号决定动作，止损距离与风险预算决定单次买入上限。
- 仓位模式会影响“动态防守仓位”：纯交易仓可降到 0%，长期底仓+交易仓会根据趋势和风险动态保留少量仓位，但不会机械死守固定底仓。
- 标的类型目前只影响数据源、估值映射、搜索识别，不影响仓位上限，也不会按类型强制封顶。

## 注意

测试只能保证规则计算逻辑一致，不能保证交易一定盈利。市场结果仍受行情、数据源准确性、滑点、停牌、跳空、重大消息等影响。
