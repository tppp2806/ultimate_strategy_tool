# v45 覆盖文件补丁：系统自算 PE 百分位不再误判 1 条数据为成功

## 覆盖文件

把本补丁内文件复制到原项目根目录覆盖：

```text
app.py
PATCH_V45_README.md
```

## 修复内容

1. 【系统自算历史百分位】只接受真正的历史估值序列。
   - 过去如果 AKShare 只返回 1 条当前估值，也会显示 `valuation:akshare_index_history_latest 成功（1条）`。
   - 但 1 条数据无法计算历史 PE 百分位，所以这次改为明确失败并继续报错说明。

2. 实时操作页选择【系统自算历史百分位】时，不再用 AKShare 当前单行估值兜底。
   - 系统自算 = 必须有历史 PE 序列。
   - 当前 1 行估值只能提供 current_pe/current_pb，不能提供历史百分位。

3. 历史序列校验更严格。
   - 默认至少需要 30 条历史估值。
   - 必须实际算出 `pe_percentile`。
   - 失败信息会写明：总条数、PE有效点、PE百分位点。

4. 日志里的 rows 不再固定写 1。
   - 历史链路成功时会显示真实历史序列条数。

## 说明

如果仍然失败，说明当前 AKShare 接口没有为该指数返回历史估值序列。此时不是脚本“算不出来”，而是数据源没有提供足够历史 PE 数据。

可优先测试：

```text
沪深300：000300 / SH000300 / 沪深300
上证50：000016
中证500：000905
中证1000：000852
创业板指：399006
科创50：000688
```

如果这些也返回历史点数不足，需要检查本机 AKShare 版本或该接口是否可用：

```bash
pip install -U akshare
```
