# v46 覆盖文件补丁：系统自算 PE 百分位日志增强

## 覆盖文件

```text
app.py
```

## 修复内容

1. 增强【系统自算历史 PE 百分位】失败日志。
   - 显示 AKShare 版本。
   - 显示指数映射结果：输入代码、指数名称、指数代码、纯数字代码。
   - 显示每次调用的 AKShare 函数与参数。
   - 显示返回行数、列名、解析后的序列条数、PE/PB有效点数量、PE/PB百分位数量。

2. 不再只显示笼统的：

```text
AKShare 未返回历史指数估值
```

会显示类似：

```text
akshare版本=...
映射结果 code=SH000300, name=沪深300, digits=000300
index_value_hist_funddb({'symbol': '沪深300', 'indicator': '市盈率'}) -> 返回0行，列=[...]
函数不存在：ak.index_value_hist_funddb
```

3. 估值链路错误展示长度从 240 字提升到 1600 字，避免关键诊断信息被截断。

## 使用方式

复制 `app.py` 到项目根目录覆盖，然后重启：

```bash
python app.py
```

然后在设置里选择【系统自算历史百分位】，重新点击【拉取所选标的数据】，查看【数据源尝试记录】。

## 说明

如果日志显示 `函数不存在：ak.index_value_hist_funddb`，说明当前 AKShare 版本没有这个接口或接口名变化。

如果日志显示返回 0 行，说明接口存在，但当前 symbol 参数无法命中历史估值数据，需要改映射名称或换接口。
