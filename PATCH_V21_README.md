# v21 修复说明

## 修复内容

1. 修复拉取状态偶发停留在“拉取中…”
   - 新增 `fetchInFlight` 防并发锁。
   - 拉取结束后强制恢复按钮文案为“拉取所选标的数据”。

2. 修复卖出/清仓类信号反向买入的问题
   - 当【跌破20日线 / 跌破50日线 / 跌破200日线 / 触发初始止损 / 止盈】等卖出类信号触发时，如果系统底仓高于当前仓位，不再为了补足底仓而建议买入。
   - 系统底仓现在只作为“最多保留到哪里”的防守下限，不是无买点时的补仓信号。

3. 当前有仓位且目标不变时，结果显示为【持有】
   - 以前可能显示【观望 维持10%】。
   - 现在有持仓且无调整时显示【持有 维持10%】。

4. 保留 v20 的量价淡化逻辑
   - 勾选【暂无明显量价信号】时，其他量价项会淡化。
   - 勾选具体量价项时，【暂无明显量价信号】会淡化并互斥。

## 设计说明

- 【入场信号】仍保持互斥：选择当前最主导的买点。
- 【减仓 / 清仓信号】仍保持互斥：选择最严重/最优先的风险项。
- 【触发初始止损】指价格已经跌到你买入前设定的初始止损价，不是跌破均线，也不是普通回调。
