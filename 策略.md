# 策略 Python 文件完整编写指南

适用项目结构：

```text
strategies/
  base.py
  registry.py
  strategy_engine.py
  factors/
    mini_factors.py
  families/
    simple_ma.py
    trend_signal_control.py
    mini_factor_timing.py
    _template.py
```

新增一个总体策略时，只需要在：

```text
strategies/families/
```

下面新增一个非下划线开头的 `.py` 文件，例如：

```text
strategies/families/my_strategy.py
```

重启 Flask 后，系统会通过 `registry.py` 自动发现该策略，不需要手动修改 `registry.py`、`strategy_engine.py` 或 `app.py`。

------

# 1. 策略文件必须导出的内容

一个策略 `.py` 文件至少必须包含：

```python
FAMILY_KEY = "my_strategy"
FAMILY_META = {...}

def target_weight(cfg, signals):
    return target, notes
```

完整推荐结构如下：

```python
from __future__ import annotations

from typing import Any, Dict, List, Tuple

from ..base import (
    clamp,
    core_asset_floor_bounds,
    core_asset_profile,
    get_strategy,
    lower_floor,
    pct2,
)

FAMILY_KEY = "my_strategy"
SIGNAL_DRIVEN = False

FAMILY_META: Dict[str, Any] = {
    "order": 90,
    "name": "我的策略",
    "short_name": "我的策略",
    "desc": "一句话说明策略逻辑。",
    "status": "研究中",
    "axes": ["趋势", "估值"],
}

STYLE_PARAM_PRESETS: Dict[str, Dict[str, Any]] = {
    ...
}

STYLE_PARAM_SCHEMA: List[Dict[str, Any]] = [
    ...
]

INPUT_SCHEMA: List[Dict[str, Any]] = [
    ...
]

def target_weight(cfg: Dict[str, Any], signals: Dict[str, Any]) -> Tuple[float, List[str]]:
    ...
```

------

# 2. 自动注册规则

`strategies/registry.py` 会自动扫描：

```text
strategies/families/*.py
```

但会忽略：

```text
__init__.py
_template.py
_开头的辅助文件.py
```

一个文件只有同时满足以下条件，才会被识别成策略：

```python
FAMILY_KEY 是 str
FAMILY_META 是 dict
target_weight 是可调用函数
```

所以新增策略时必须保证：

```python
FAMILY_KEY = "unique_strategy_key"
FAMILY_META = {...}

def target_weight(cfg, signals):
    ...
```

------

# 3. FAMILY_KEY 编写规范

`FAMILY_KEY` 是策略唯一标识。

建议：

```python
FAMILY_KEY = "simple_ma"
FAMILY_KEY = "mini_factor_timing"
FAMILY_KEY = "valuation_trend_mix"
```

要求：

1. 只能用小写英文、数字、下划线。
2. 不要用中文。
3. 不要和已有策略重复。
4. 最好和文件名一致。

例如：

```text
文件名：valuation_trend_mix.py
FAMILY_KEY = "valuation_trend_mix"
```

------

# 4. SIGNAL_DRIVEN 的含义

```python
SIGNAL_DRIVEN = False
```

这是非常重要的开关。

## 4.1 推荐默认值

新写的大多数策略都应该使用：

```python
SIGNAL_DRIVEN = False
```

含义是：

不管是【交易模式】还是【定投增强模式】，系统都会调用本策略自己的：

```python
target_weight(cfg, signals)
```

也就是说，策略完全由自己的 Python 文件控制。

## 4.2 什么时候才用 True

只有当策略明确想沿用旧版趋势信号硬规则时，才设置：

```python
SIGNAL_DRIVEN = True
```

注意：

如果 `SIGNAL_DRIVEN = True`，在纯交易仓模式下，系统可能不会完全使用本策略的 `target_weight()`，而是走旧的趋势信号硬规则分支。

所以新策略一般不要设置 True。

推荐：

```python
SIGNAL_DRIVEN = False
```

# 5. FAMILY_META 字段说明

示例：

```python
FAMILY_META: Dict[str, Any] = {
    "order": 40,
    "name": "估值趋势混合策略",
    "short_name": "估值趋势",
    "desc": "结合估值百分位和中长期趋势，生成目标仓位。",
    "status": "研究中",
    "axes": ["估值", "趋势", "风控"],
}
```

字段说明：

| 字段         | 必填 | 说明                                      |
| ------------ | ---- | ----------------------------------------- |
| `order`      | 是   | 前端策略排序，数字越小越靠前              |
| `name`       | 是   | 完整策略名称                              |
| `short_name` | 否   | 短名称，用于窄区域显示                    |
| `desc`       | 是   | 策略说明                                  |
| `status`     | 否   | 例如：`研究中`、`可回测`、`可实盘/可回测` |
| `axes`       | 否   | 策略使用的主要维度，如 `["趋势", "估值"]` |

# 6. target_weight 的核心约定

每个策略必须实现：

```python
def target_weight(cfg: Dict[str, Any], signals: Dict[str, Any]) -> Tuple[float, List[str]]:
    ...
```

返回值：

```python
return target, notes
```

其中：

```python
target
```

是目标仓位，单位是 `0~1`。

例如：

```python
0.00  # 空仓
0.25  # 25%仓位
0.60  # 60%仓位
0.92  # 92%仓位
notes
```

是解释文本列表。

例如：

```python
notes = [
    "估值处于低位，允许提高仓位。",
    "价格仍在200日线上方，中期趋势未破坏。",
]
```

前端会显示这些解释。

------

# 7. target_weight 必须返回“目标仓位”，不是买卖比例

错误示例：

```python
return 0.15, ["买入15%"]
```

这会被系统理解成“目标仓位 15%”，不是“加仓 15%”。

正确做法：

如果策略本身是目标仓位模型，直接返回目标仓位：

```python
target = 0.60
return target, ["策略目标仓位为60%。"]
```

如果策略本身是加减仓模型，必须先读取当前仓位，再转换成目标仓位：

```python
cur = _current_position(cfg)
delta = 0.15
target = clamp(cur + delta, 0.0, 1.0)
return target, ["当前信号触发加仓。"]
```

------

# 8. 当前仓位读取方式

策略文件不能直接 import `app.py` 里的 `current_position()`。

如果策略需要知道当前仓位，可以在策略文件里写一个本地函数：

```python
def _current_position(cfg: Dict[str, Any]) -> float:
    try:
        plan = max(float(cfg.get("plan_amount") or 0.0), 0.0)
        market_amount = max(float(cfg.get("current_position_amount") or 0.0), 0.0)
        profit_pct = float(cfg.get("current_profit_pct") or 0.0)
        profit_factor = max(1.0 + profit_pct / 100.0, 0.0001)
        cost_amount = market_amount / profit_factor
        return clamp(cost_amount / plan, 0.0, 0.9999) if plan > 0 else 0.0
    except Exception:
        return 0.0
```

这个口径和主程序一致：

```text
当前仓位 = 当前持仓市值 / (1 + 持仓盈亏率) / 计划资金
```

也就是按“已占用计划本金 / 计划资金”计算，而不是直接按当前市值计算。

# 9. 常用 base.py 工具函数

策略文件可以从 `..base` 导入：

```python
from ..base import (
    clamp,
    pct2,
    get_strategy,
    core_asset_profile,
    core_asset_floor_bounds,
    lower_floor,
    _as_optional_pct_value,
)
```

## 9.1 clamp

```python
clamp(x, low, high)
```

把数值限制在区间内。

```python
target = clamp(target, 0.0, 1.0)
```

## 9.2 pct2

```python
pct2(0.236)
```

返回：

```text
23.60%
```

适合写解释文本。

## 9.3 get_strategy

```python
style = get_strategy(cfg)
```

读取当前参数风格的有效参数。

例如：

```python
risk_mult = float(style.get("risk_multiplier", 1.0))
buy_step = float(style.get("my_buy_pct", 20.0)) / 100.0
```

注意：

`STYLE_PARAM_SCHEMA` 里声明的 `_pct` 字段，在 `style` 里仍然是 `0~100` 的百分数，不是 `0~1`。

所以策略内部使用时通常要 `/ 100.0`。

例如：

```python
valuation_weight = float(style.get("valuation_weight_pct", 100.0)) / 100.0
```

## 9.4 core_asset_profile

```python
profile = core_asset_profile(cfg)
```

返回：

```python
"core"  # 定投增强策略
""      # 纯交易模式
```

判断逻辑：

```python
cfg.get("position_mode") == "core_satellite"
```

就是定投增强策略。

## 9.5 core_asset_floor_bounds

```python
low, high = core_asset_floor_bounds(core_asset_profile(cfg), cfg)
```

返回当前仓位模式允许的最低、最高仓位。

单位是 `0~1`。

例如：

```python
low = 0.05
high = 0.92
```

策略最后通常应该：

```python
target = clamp(target, low, high)
```

## 9.6 lower_floor

```python
floor = lower_floor(cfg, signals)
```

返回系统自动计算的动态防守底仓。

注意：

不是所有策略都必须调用它。

如果策略强调“纯净规则”，例如简易均线策略，只想读取 `ma_position`，那就不要调用 `lower_floor()`，因为它会参考趋势、估值、风险等信息。

原则：

```text
纯策略：只读取自己声明要读的字段。
综合策略：可以使用 lower_floor。
```

# 10. 参数风格：STYLE_PARAM_PRESETS

每个策略可以定义三套默认参数：

```python
STYLE_PARAM_PRESETS: Dict[str, Dict[str, Any]] = {
    "balanced": {...},
    "defensive": {...},
    "aggressive": {...},
}
```

这三套不是“总体策略”，只是同一个策略下的执行风格。

中文含义：

```text
defensive  = 防守
balanced   = 均衡
aggressive = 进攻
```

推荐每个策略都提供这三套。

# 15. 参数偏离规则

当前系统的参数风格逻辑是：

```text
均衡 = 基准
防守/进攻 = 基于均衡做偏离计算
```

字段是否参与偏离，主要由字段名决定。

## 15.1 阈值类字段

字段名包含以下关键词时，通常会被当成“阈值/仓位边界”：

```text
cap
limit
threshold
above
below
stop
floor
min_
max_
dca_base_buy
core_step
core_base
```

例如：

```python
bear_cap_pct
below200_cap_pct
high_valuation_cap_pct
buy_step_limit_pct
sell_step_limit_pct
strict_max_position_pct
```

## 15.2 幅度类字段

字段名以 `_pct` 结尾，但不属于阈值类，通常会被当成“操作/权重幅度”。

例如：

```python
trend_weight_pct
valuation_weight_pct
entry_bonus_pct
risk_penalty_pct
quality_weight_pct
```

## 15.3 不想参与偏离

不要使用 `_pct` 结尾，或者使用 `select`、`checkbox`、普通倍率字段。

例如：

```python
risk_multiplier
mode
enabled
```

# 17. 自定义 checkbox 的安全读取

对于自定义 checkbox，建议不要直接：

```python
if signals.get("my_risk_flag"):
    ...
```

因为表单可能传入字符串。

推荐写一个安全函数：

```python
def _bool_signal(signals: Dict[str, Any], key: str, default: bool = False) -> bool:
    value = signals.get(key, default)
    if isinstance(value, str):
        return value.strip().lower() not in {"", "0", "false", "off", "no", "none"}
    return bool(value)
```

使用：

```python
if _bool_signal(signals, "my_risk_flag"):
    target -= 0.10
```

# 18. signals 中可直接读取的通用字段

`signals` 是策略最重要的输入。

策略只应该读取自己需要的字段。

不要为了“保险”读取所有字段，否则策略会变得不纯净。

------

## 18.1 趋势环境字段

```python
signals["market_state"]
```

可能值：

| 值            | 含义                         |
| ------------- | ---------------------------- |
| `bear`        | 200日线下方且200日线向下     |
| `below_200`   | 价格未站上200日线            |
| `sideways`    | 震荡                         |
| `above_200`   | 价格站上200日线              |
| `strong_bull` | 50日线 > 200日线，且价格强势 |

读取方式：

```python
market = str(signals.get("market_state", "sideways") or "sideways")
```

------

## 18.2 入场状态字段

```python
signals["entry_state"]
```

可能值：

| 值                  | 含义               |
| ------------------- | ------------------ |
| `none`              | 无入场信号         |
| `reversal_50`       | 站回50日线反转试仓 |
| `breakout`          | 平台/前高突破      |
| `pullback_hold`     | 回踩20/50日线不破  |
| `continuation_high` | 强趋势持续创新高   |

# 20. 自动行情技术因子

以下字段由自动行情数据或回测数据生成。

没有数据时可能为：

```python
None
```

策略必须能处理 None。

推荐写：

```python
def _num(signals: Dict[str, Any], key: str):
    raw = signals.get(key)
    if raw is None or raw == "":
        return None
    try:
        value = float(raw)
    except (TypeError, ValueError):
        return None
    if value != value or value in (float("inf"), float("-inf")):
        return None
    return value
```

# 24. 策略解释 notes 的原则

`notes` 应该写“定性理由”，不要写完整计算过程。

推荐：

```python
notes.append("估值处在低位，允许提高基础仓位。")
notes.append("价格仍在200日线上方，中期趋势未破坏。")
```

不推荐：

```python
notes.append("target = 0.5 + 0.08 - 0.03 = 0.55")
```

原因：

前端指标区会展示数值，解释区只需要讲为什么。

------

# 25. 策略纯净性原则

每个策略必须明确自己读取哪些字段。

例如，简易均线策略只读取：

```python
signals["ma_position"]
```

就不要读取：

```python
pe_percentile
roe_pct
market_state
volume_confirm
market_risk
```

否则策略就不纯净，用户切换策略时会觉得不同策略本质一样。

推荐在文件顶部 docstring 写清楚：

```python
"""简易均线策略。

纯净规则：
只读取 ma_position。
不读取估值、ROE、大盘趋势、量能、止损、市场风险。
"""
```

综合策略可以读取多个因子，但也要写清楚。

------

# 26. 缺失数据处理原则

任何自动因子都可能是 None。

错误写法：

```python
if signals["return_60d"] > 10:
    ...
```

正确写法：

```python
ret60 = _num(signals, "return_60d")
if ret60 is not None and ret60 > 10:
    ...
```

不要因为缺少某个因子就报错。

缺失时应该按中性处理。

------

# 27. 回测兼容原则

策略必须满足：

```text
同一份 target_weight 同时支持实时计算和历史回测。
```

要求：

1. 不要访问网络。
2. 不要读取本地文件。
3. 不要依赖当前日期。
4. 不要调用随机数。
5. 不要修改全局状态。
6. 不要 import app.py。
7. 只根据 cfg 和 signals 计算结果。

正确：

```python
def target_weight(cfg, signals):
    ...
```

错误：

```python
def target_weight(cfg, signals):
    requests.get(...)
    datetime.now()
    open("xxx.csv")
```

# 28. 新增自动因子的规则

如果策略需要一个已经存在的因子，直接从 `signals` 读取。

如果策略需要一个不存在的因子，有两种方式：

## 28.1 手动输入因子

适合：

```text

临时信号
AI不能自动从行情算出的状态
```

## 28.2 自动行情因子

如果要让系统自动生成新因子，需要修改：

```text
app.py
```

中的：

```python
compute_indicators()
parse_signals()
enrich_history_records_for_cache()
```

策略文件本身不能凭空读取一个不存在的自动因子。

```python
def _num(signals: Dict[str, Any], key: str) -> Optional[float]:
    raw = signals.get(key)
    if raw is None or raw == "":
        return None
    try:
        value = float(raw)
    except (TypeError, ValueError):
        return None
    if value != value or value in (float("inf"), float("-inf")):
        return None
    return value


def _pct_param(style: Dict[str, Any], key: str, default: float) -> float:
    try:
        return clamp(float(style.get(key, default)), 0.0, 500.0) / 100.0
    except (TypeError, ValueError):
        return default / 100.0


def _cap_param(style: Dict[str, Any], key: str, default: float) -> float:
    try:
        return clamp(float(style.get(key, default)), 0.0, 100.0) / 100.0
    except (TypeError, ValueError):
        return default / 100.0


def _manual_bias(signals: Dict[str, Any], key: str) -> int:
    raw = str(signals.get(key, "0") or "0").strip().lower()
    if raw in {"-1", "0", "1"}:
        return int(raw)
    return 0


def target_weight(cfg: Dict[str, Any], signals: Dict[str, Any]) -> Tuple[float, List[str]]:
    """返回目标仓位 0~1 和解释文本列表。"""
    style = get_strategy(cfg)
    notes: List[str] = []

    market = str(signals.get("market_state", "sideways") or "sideways")
    exit_state = str(signals.get("exit_state", "none") or "none")

    trend_weight = _pct_param(style, "trend_weight_pct", 100.0)
    valuation_weight = _pct_param(style, "valuation_weight_pct", 100.0)
    quality_weight = _pct_param(style, "quality_weight_pct", 100.0)
    vol_penalty = _pct_param(style, "volatility_penalty_pct", 100.0)
    risk_multiplier = clamp(float(style.get("risk_multiplier", 1.0) or 1.0), 0.1, 5.0)

    base_table = style.get("core_base") or {
        "bear": 0.12,
        "below_200": 0.26,
        "sideways": 0.50,
        "above_200": 0.70,
        "strong_bull": 0.82,
    }

    target = float(base_table.get(market, 0.50))

    if market in {"above_200", "strong_bull"}:
        notes.append("中长期趋势偏多，允许维持较高基础仓位。")
    elif market in {"bear", "below_200"}:
        notes.append("价格处于长期趋势防守区，基础仓位需要降低。")
    else:
        notes.append("震荡环境方向不明确，基础仓位保持中性。")

    ret60 = _num(signals, "return_60d")
    if ret60 is not None:
        if ret60 >= 12:
            target += 0.05 * trend_weight
            notes.append("60日动量较强，趋势因子提供正向支持。")
        elif ret60 <= -10:
            target -= 0.06 * trend_weight
            notes.append("60日动量偏弱，趋势因子压低仓位。")

    pe = _as_optional_pct_value(signals.get("pe_percentile"))
    pb = _as_optional_pct_value(signals.get("pb_percentile"))

    if pe is not None:
        if pe <= 30:
            target += (30.0 - pe) / 30.0 * 0.10 * valuation_weight
            notes.append("PE百分位偏低，估值具备安全边际。")
        elif pe >= 85:
            target -= (pe - 85.0) / 15.0 * 0.14 * valuation_weight
            notes.append("PE百分位偏高，新增仓位需要降速。")
        else:
            notes.append("PE百分位处于中性区间，对仓位影响有限。")

    if pb is not None:
        if pb <= 35:
            target += (35.0 - pb) / 35.0 * 0.04 * valuation_weight
            notes.append("PB百分位偏低，提供轻微估值支持。")
        elif pb >= 85:
            target -= (pb - 85.0) / 15.0 * 0.06 * valuation_weight
            notes.append("PB百分位偏高，进一步压低进攻性。")

    roe_raw = signals.get("roe_pct")
    try:
        roe = float(roe_raw) if roe_raw not in (None, "") else None
    except (TypeError, ValueError):
        roe = None

    if roe is not None:
        if roe >= 18:
            target += 0.06 * quality_weight
            notes.append("ROE表现优秀，质量因子支持长期持有。")
        elif roe >= 12:
            target += 0.03 * quality_weight
            notes.append("ROE达到基本质量要求。")
        elif roe < 8:
            target -= 0.05 * quality_weight
            notes.append("ROE偏弱，不适合提高进攻性。")

    vol60 = _num(signals, "volatility_60d")
    drawdown = _num(signals, "drawdown_252d")

    if vol60 is not None and vol60 >= 35:
        target -= min((vol60 - 35.0) / 30.0, 1.0) * 0.08 * vol_penalty
        notes.append("60日波动率偏高，需要压低目标仓位。")

    if drawdown is not None:
        if drawdown <= -30 and market in {"bear", "below_200"}:
            target -= 0.08 * vol_penalty
            notes.append("深度回撤尚未完成趋势修复，暂不盲目抄底。")
        elif -25 <= drawdown <= -10 and market in {"above_200", "strong_bull"}:
            target += 0.04 * vol_penalty
            notes.append("回撤后趋势仍未破坏，具备一定修复吸引力。")

    if signals.get("market_risk"):
        target -= 0.10 * vol_penalty
        notes.append("大盘或同类资产同步走弱，策略降低风险暴露。")

    if exit_state in {"below_50", "failed_breakout"}:
        target -= 0.10
        notes.append("中期结构转弱或突破失败，需要降低交易仓。")
    elif exit_state in {"below_200", "hit_stop"}:
        target -= 0.16
        notes.append("长期趋势或初始止损信号触发，仓位进入防守状态。")

    manual = _manual_bias(signals, "vt_manual_bias")
    if manual > 0:
        target += 0.04
        notes.append("手动修正偏多，轻微提高目标仓位。")
    elif manual < 0:
        target -= 0.04
        notes.append("手动修正偏空，轻微降低目标仓位。")

    target = 0.50 + (target - 0.50) * clamp(risk_multiplier, 0.65, 1.35)

    floor = lower_floor(cfg, signals)
    low, high = core_asset_floor_bounds(core_asset_profile(cfg), cfg)

    cap = high
    if market == "bear":
        cap = min(cap, _cap_param(style, "bear_cap_pct", 28.0))
    elif market == "below_200":
        cap = min(cap, _cap_param(style, "below200_cap_pct", 45.0))

    if pe is not None and pe >= 95:
        cap = min(cap, _cap_param(style, "extreme_valuation_cap_pct", 58.0))
    elif pe is not None and pe >= 90:
        cap = min(cap, _cap_param(style, "high_valuation_cap_pct", 70.0))

    target = clamp(max(target, floor), low, cap)

    if target >= 0.68:
        regime = "估值趋势偏多"
    elif target <= 0.32:
        regime = "估值趋势偏空"
    else:
        regime = "估值趋势中性"

    signals["strategy_match_label"] = f"估值趋势混合：{regime}"
    signals["strategy_confidence"] = int(clamp(58 + abs(target - 0.50) * 55, 55, 86))

    if not notes:
        notes.append("当前因子未形成明确方向，策略维持中性仓位。")

    return target, notes
```

# 30. 纯加减仓策略模板

如果策略不是直接给目标仓位，而是根据某个信号加仓/减仓，可以使用这种结构。

示例：

```python
def target_weight(cfg: Dict[str, Any], signals: Dict[str, Any]) -> Tuple[float, List[str]]:
    style = get_strategy(cfg)
    cur = _current_position(cfg)

    signal = str(signals.get("my_signal", "hold") or "hold")

    if signal == "buy":
        delta = float(style.get("buy_pct", 20.0)) / 100.0
        note = "信号触发买入。"
    elif signal == "sell":
        delta = -float(style.get("sell_pct", 20.0)) / 100.0
        note = "信号触发卖出。"
    else:
        delta = 0.0
        note = "信号中性，维持当前仓位。"

    low, high = core_asset_floor_bounds(core_asset_profile(cfg), cfg)
    target = clamp(cur + delta, low, high)

    signals["strategy_match_label"] = "我的策略：加减仓信号"
    signals["strategy_confidence"] = 68

    return target, [note]
```

重点：

```python
target = cur + delta
```

不要直接返回 `delta`。

# 31. 策略文件禁止事项

不要这样写：

```python
import requests

from app import current_position
```

不要在策略里：

```text
访问网络
读写文件
修改 app.py 全局变量
使用随机数
使用当前真实日期
依赖前端 DOM
依赖用户本地路径
```

策略文件只负责：

```text
cfg + signals -> target_weight + notes
```
