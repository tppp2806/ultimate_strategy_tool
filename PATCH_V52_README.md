# v52 覆盖补丁：修复回测缓存与蛋卷估值日期对齐

覆盖文件：

- `app.py`
- `static/app.js`

## 修复内容

1. 修复【回测配置】刷新后重置的问题。
   - 原因：前端使用了 `BACKTEST_KEY`，但常量没有定义，保存/读取 localStorage 时直接失败。
   - 现在新增 `trendRisk.backtestState.v3`，回测表单会正常持久化。

2. 修复蛋卷历史估值最新日期显示慢一天的问题。
   - 原因：蛋卷 `ts` 是按北京时间交易日 00:00 给出，旧代码按 UTC 解析会变成前一天。
   - 现在蛋卷 `ts` 固定按 UTC+8 转日期，因此 `1780588800000` 会解析为 `2026-06-05`，与 detail 里的 `date: "06-05"` 对齐。

## 覆盖后验证

1. 刷新页面，修改回测配置，重新刷新，配置应保留。
2. 使用蛋卷估值接口时，估值序列最新日期应与页面 detail 日期对齐。
