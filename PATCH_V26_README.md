# V26 覆盖文件补丁：解释标点修复 + 计算逻辑自测修正

把本补丁包里的文件复制到原项目根目录，选择覆盖：

app.py

## 改动

1. 修复解释文案里出现 `。；` 的问题
   - 现在多条解释会统一清理句尾标点后再拼接。
   - 示例：
     - 旧：`初始止损被触发，交易失败，先退出交易仓。；当前仓位...`
     - 新：`初始止损被触发，交易失败，先退出交易仓；当前仓位...。`

2. 修复【跌破20日线】在“当前仓位低于系统底仓/防守目标”时被后续买入逻辑覆盖的问题
   - 旧逻辑：当前仓位低于底仓时，【跌破20日线】可能被后续【强趋势但无新买点】覆盖，解释不一致。
   - 新逻辑：保留【跌破20日线】作为命中规则，但不追加卖出，也不因底仓差额买入。

3. 清理一处重复字段
   - 删除 `valuation_adjustment` 重复赋值。

## 自测覆盖

已用后端 `compute_decision()` 做了以下场景测试：

| 场景 | 结果 |
|---|---|
| 长期底仓+交易仓；强多头；无买点；仓位10% | 持有，维持10% |
| 纯交易仓；强多头；无买点；仓位10% | 持有，维持10% |
| 长期底仓+交易仓；触发初始止损；当前仓位低于系统底仓 | 持有，维持当前仓位；不补底仓、不追加卖出 |
| 纯交易仓；触发初始止损 | 清仓，目标0% |
| 长期底仓+交易仓；跌破20日线；仓位50% | 减仓，降到系统防守目标附近 |
| 长期底仓+交易仓；跌破20日线；仓位10%且低于系统底仓 | 持有，维持10%；命中规则仍显示跌破20日线 |
| 纯交易仓；跌破20日线 | 按交易仓纪律减仓，必要时可降到0 |
| 强多头回踩不破；仓位50% | 加仓，显示本次变化量 |
| `stock / etf / fund / index` 使用同一组信号 | 决策结果一致，不按标的类型设置仓位上限 |

## 标的类型说明

- `asset_kind` 现在主要用于：数据源选择、代码/指数映射、估值映射、行情兜底路径。
- 仓位计算不再按 `stock / etf / fund / index` 设置上限；计划资金就是 100% 上限。
- 具体决策差异主要来自【仓位模式】：
  - 长期底仓 + 交易仓：系统自动计算底仓，破位时不会低于底仓继续卖，也不会因为低于底仓而强行买。
  - 纯交易仓：底仓为0，破位/止损可以降到0。
