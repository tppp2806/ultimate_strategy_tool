## 主要改动

1. 设置页新增【PE百分位来源】
   - 自动：系统自算优先，蛋卷兜底
   - 系统自算历史百分位
   - 蛋卷估值优先

2. 实时操作页自动获取【历史 PE 百分位 %】时会遵守上面的来源设置
   - 系统自算：使用可获取的 PE/PB 历史序列计算当前分位
   - 蛋卷：优先请求 `GET /djapi/index_eva/detail/{code}`

3. 历史回测新增【估值参与方式】
   - 不使用估值修正：默认，适合先验证纯趋势/风控逻辑
   - 使用固定估值假设：PE/PB/ROE 会作为整段回测固定输入
   - 使用历史估值序列：尝试获取逐日历史 PE/PB，并且只用当日及以前数据计算百分位，避免未来函数

4. 历史估值序列支持范围
   - 国内宽基指数及映射到这些指数的 ETF / 联接基金：优先 AKShare FundDB 历史估值
   - A股个股：尝试 AKShare / 乐咕历史估值
   - 美股个股：暂不强行自算历史估值，避免用不完整财报数据污染回测

## 注意

历史估值序列需要本地网络和 AKShare 接口可用。若获取失败，回测会自动退回“不使用估值修正”，并在回测结果的备用链路失败记录里显示原因。
