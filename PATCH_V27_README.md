## 改动

1. 【长期底仓 + 交易仓】不再机械坚守 40% 左右底仓。
   - 旧逻辑容易把强多头基础底仓抬到 40%。
   - 新逻辑改为【动态防守仓位】，会随趋势、估值、ROE、破位状态和大盘/板块风险自动下降。

2. 弱势行情下防守仓位会明显降低。
   - 200日线下方且向下：可接近 0%。
   - 未站上200日线：通常 0%~10%。
   - 跌破200日线：通常 0%~8%，高估/市场风险时可到 0%~3%。
   - 触发初始止损：通常不高于 5%，高估/弱势/市场风险时可到 0%。

3. 文案统一。
   - 结果指标从【系统底仓保留】改成【系统防守仓位】。
   - 仓位模式说明从【底仓系统自动计算】改成【防守仓位动态计算】。
   - 解释文案里的“底仓差额买入”改成“防守仓位差额买入”。

4. 计划资金仍然是 100% 上限。
   - 本次没有恢复任何按标的类型封顶的逻辑。
   - stock / etf / fund / index 仍只影响数据源和估值映射，不影响仓位上限。

## 自测场景

已测试：

- 长期底仓+交易仓，强多头，无买点，仓位10%：持有，维持10%。
- 长期底仓+交易仓，强多头，高PE，无买点，仓位10%：持有，维持10%。
- 长期底仓+交易仓，触发初始止损，高估，仓位50%：清交易仓，目标可降到0%。
- 纯交易仓，触发初始止损：清仓到0%。
- 跌破200日线且高估：动态防守仓位可降到0%。
- 不同标的类型 stock / etf / fund / index：同一交易信号下仓位结果一致，不按类型封顶。
