# v31 覆盖文件补丁：选项名称与悬浮注释直白化

## 覆盖方式

把本补丁中的文件复制到原项目根目录覆盖：

```text
app.py
templates/index.html
static/app.js
```

## 主要改动

1. 重新命名容易误解的选项
   - 【平台/前高突破】改为【收盘突破60日前高】。
   - 【强趋势持续创新高】改为【强趋势中收盘突破20日新高】。
   - 【触发初始止损】改为【跌到初始止损线】。
   - 【回踩20/50日线不破】改为【回踩20/50日线后收住】。
   - 【突破失败】改为【突破后收盘跌回】。
   - 【暂无买点 / 暂无破位 / 无明显盈利】分别改为更直白的状态描述。

2. 重写悬浮注释
   - 明确“收盘突破60日前高”是最近60个交易日最高收盘价，不是复杂平台形态识别。
   - 明确“强趋势中收盘突破20日新高”是趋势延续，不等于任何位置都能追。
   - 明确“跌到初始止损线”只在持仓亏损达到止损距离时成立，不等同于仓位超过风险预算。
   - 明确量价辅助只做加减权重，不单独主导买卖。

3. 同步结果解释文案
   - 后端命中规则与解释文字也改成更直白的表述。
   - 避免前端选项和右侧解释名称不一致。

## 未改动

- 没有修改字段值，例如 `entry_state=breakout`、`exit_state=hit_stop` 等仍保持不变。
- 没有修改仓位计算公式。
- 没有修改数据源逻辑。
