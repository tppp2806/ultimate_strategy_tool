## 核心改动

### 1. 新增「信号质量」

- 不是历史胜率，也不是预测概率。
- 用趋势环境、主入场/退出信号、量价确认、PE/PB、ROE、市场风险综合计算。
- 买入类信号低于一定质量时，会降低新增仓位；质量太差时不主动新增交易仓。

### 2. 新增「预期赔率R」

- 没有输入目标价/压力位时，只能做规则估算。
- 以不同入场类型为基础：
  - 回踩20/50日线后收住：赔率较好；
  - 收盘突破60日前高：中等偏好；
  - 强趋势中收盘突破20日新高：容易追高，默认更克制；
  - 站回50日线但未站上200日线：只适合试仓。
- 高PE、低ROE、远离20日线、长上影、未站稳关键位都会降低赔率。
- 预期赔率低于1R时，不新增交易仓。

### 3. 新增「操作频率」

- 把不同信号标成：低频优先买点 / 中低频确认买点 / 中频突破试错 / 偏高频追涨 / 风险退出等。
- 高频噪音信号会降低新增仓位，避免脚本过度交易。

### 4. 卖出上限动态化

- 轻微跌破20日线：卖出更克制；
- 跌破50日线：卖出上限提高；
- 跌破200日线 / 跌到初始止损线：允许快速退出交易仓；
- 高PE、低ROE、长上影、未站稳、大盘走弱只在已有卖出信号时放大卖出上限，不会单独触发卖出。

### 5. 右侧指标新增

新增显示：

```text
信号质量
预期赔率
操作频率
```

## 设计原则

这版不是把指标越堆越多，而是加入三道系统级过滤：

```text
信号质量：这次信号是否足够干净
预期赔率：这次交易是否值得冒风险
操作频率：这是不是容易过度交易的噪音机会
```

它们只限制新增仓位，不替代主规则。
